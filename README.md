# Yaran

Yaran is a terminal address book manager written in Go. It stores contacts in
SQLite and provides a keyboard-driven TUI for browsing, editing, filtering,
importing, and exporting address data.

The name `yaran` is Persian for `dearest friends`.

## Features

- Persistent SQLite storage with automatic schema initialization
- Add, edit, delete, and inspect address records
- Single-field filtering from the TUI
- Import and export support for `csv`, `json`, `html`, and `vcard`
- Progress feedback for import and export operations
- Duplicate protection and birthday validation

## Requirements

- Go `1.25+`

## Run

Start the app:

```bash
go run ./cmd/yaran
```

Build a local binary:

```bash
go build -o yaran ./cmd/yaran
./yaran
```

Use a custom database path:

```bash
go run ./cmd/yaran --db-path /path/to/yaran.db
```

Show the version:

```bash
go run ./cmd/yaran --version
```

By default, Yaran stores data in `~/.yaran.db`.

## TUI Usage

The main screen shows the address table, the selected record details, current
result count, filter controls, and import/export actions.

Keyboard shortcuts:

- `a` add an address
- `e` edit the selected address
- `d` delete the selected address
- `Enter` open the selected address details
- `/` or `f` focus the filter input
- `Tab` and `Shift+Tab` cycle focus
- `i` open import
- `x` open export
- `r` refresh the table
- `q` or `Ctrl+C` quit

## Data Model

Each address record supports these fields:

- `name`
- `email`
- `birthday`
- `address`
- `phone`
- `mobile`
- `custom`
- `notes`

Rules:

- `name` and `email` are required
- `birthday` is optional, but when set it must use `YYYY-MM-DD`
- duplicate records are rejected

## Import and Export Formats

Supported formats:

- `csv`
- `json`
- `html`
- `vcard`

Format details:

- `csv` uses a header row with the fields `name,email,birthday,address,phone,mobile,custom,notes`
- `json` uses an array of objects with the same field names
- `html` import is intended for the HTML format generated by Yaran itself, using one `<li>` per address with `data-*` attributes
- `vcard` exports vCard `4.0` and maps `custom` values to `CATEGORIES`

Import behavior:

- records missing required fields are skipped
- records with invalid birthdays are skipped
- duplicates are skipped
